# S3単体の静的ウェブサイトホスティング手順

この手順は、CRUD LaunchpadをAmazon S3単体で静的ウェブサイトとして公開するためのものです。
S3単体公開は構成が簡単ですが、S3ウェブサイトエンドポイントはHTTPSに対応しません。

## 前提

- AWSアカウントが作成済みである
- AWSマネジメントコンソールへサインインできる
- このリポジトリの静的ファイルが手元にある

公開に必要なファイル:

```text
index.html
styles.css
script.js
assets/icon.svg
```

## 手順

1. AWSマネジメントコンソールでS3を開く
2. `バケットを作成` を選ぶ
3. バケット名を入力する
   - バケット名はグローバルで一意にする
   - 例: `crud-launchpad-sample-<任意の文字列>`
4. リージョンを選ぶ
   - 学習用なら普段使うリージョンでよい
5. バケットを作成する
6. 作成したバケットを開く
7. `プロパティ` タブを開く
8. `静的ウェブサイトホスティング` を編集する
9. 静的ウェブサイトホスティングを有効化する
10. インデックスドキュメントに `index.html` を指定する
11. 変更を保存する
12. `アクセス許可` タブを開く
13. `ブロックパブリックアクセス` を編集する
14. このバケットの公開読み取りを許可できるように設定する
15. バケットポリシーで公開読み取りを許可する
16. `オブジェクト` タブで静的ファイルをアップロードする
17. `プロパティ` タブの静的ウェブサイトホスティング欄に表示されるエンドポイントを開く

## バケットポリシー例

`<bucket-name>` は作成したバケット名に置き換えます。

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::<bucket-name>/*"
    }
  ]
}
```

## 公開URL

S3ウェブサイトエンドポイントはリージョンによって形式が異なります。
代表例:

```text
http://<bucket-name>.s3-website-<region>.amazonaws.com
http://<bucket-name>.s3-website.<region>.amazonaws.com
```

## 注意点

- S3ウェブサイトエンドポイントはHTTPSに対応しない
- 公開するにはオブジェクトをパブリックに読める必要がある
- 本番向けにはS3 + CloudFront構成を優先する
- 作業後に不要になったバケットは削除する

## 確認ポイント

- `index.html` が表示されること
- CSSとJavaScriptが読み込まれること
- `assets/icon.svg` が表示されること
- 文章の登録、検索、更新、削除が動くこと
- リロード後もlocalStorageのデータが残ること
