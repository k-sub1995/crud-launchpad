# CRUD Launchpad 仕様

## 目的

GitHub Pages や AWS など、静的サイト公開先の比較に使うためのサンプルサイトです。
ビルド処理やサーバー処理を使わず、HTML / CSS / JavaScript だけで動作します。

## 画面構成

- `index.html`: サイト本体
- `styles.css`: 画面スタイル
- `script.js`: localStorage を使った文章 CRUD
- `assets/icon.svg`: ヘッダー用の静的画像

画面上部にタイトルと固定メッセージを表示します。

## 機能

### 文章登録

文章入力欄へ本文を入力し、`登録` ボタンで保存します。
空白だけの文章は登録しません。

### 文章検索

検索欄へキーワードを入力すると、登録済み文章の本文を部分一致で絞り込みます。
大文字小文字は区別しません。

### 登録文章更新

登録済み文章の `更新` ボタンを押すと、入力欄へ文章を読み込みます。
内容を変更して `更新` ボタンを押すと、同じ文章レコードを上書きします。

### 登録文章削除

登録済み文章の `削除` ボタンを押すと、その文章を localStorage から削除します。

## 保存仕様

ブラウザの localStorage に次のキーで保存します。

```text
static-launchpad.entries.v1
```

保存値は JSON 配列です。

```json
[
  {
    "id": "generated-id",
    "text": "登録した文章",
    "createdAt": "2026-06-20T00:00:00.000Z",
    "updatedAt": "2026-06-20T00:00:00.000Z"
  }
]
```

## 動作条件

- モダンブラウザで動作します。
- JavaScript が有効である必要があります。
- localStorage が利用できる同一オリジン内で文章が保持されます。
- `file://` で直接開いた場合も基本機能は動作します。

## 公開時の確認観点

- GitHub Pages と AWS で `index.html` が初期ページとして表示されること
- `styles.css`、`script.js`、`assets/icon.svg` が相対パスで読み込めること
- 文章の登録、検索、更新、削除が公開先ごとのURL上で独立して保存されること
- ページをリロードしても登録文章が残ること
